## 1. 실습 목표
이번 실습에서는 GitHub 저장소의 첫인상이자 사용 설명서인 README.md 파일을 직접 작성해 봅니다.

### 실습 후 목표
1. README가 왜 필요한지 설명할 수 있다.
2. Markdown 문법으로 문서를 구조화할 수 있다.
3. 좋은 README에 들어가야 할 핵심 항목을 안다.
4. 자신의 프로젝트에 맞는 README를 작성하고 GitHub에 올릴 수 있다.

---

## 2. README란 무엇인가
README는 저장소를 처음 방문한 사람이 가장 먼저 읽는 문서입니다.

### 핵심 3가지
- 무엇을 하는가
- 왜 만들었는가
- 어떻게 사용하는가
